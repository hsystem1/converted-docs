**Reserva não integrada no Hunit (Channel Manager)**

**1. Sobre reserva não integrada**

Se sua reserva não apareceu automaticamente no sistema HUNIT, isso pode significar que houve uma falha na integração entre o canal de vendas e o channel manager.

Este guia vai ajudar você a identificar a causa do problema e fornecer as informações necessárias para que o suporte possa resolver a situação.

**2. Validações Iniciais (Verifique Antes de Solicitar Suporte)**

Antes de entrar em contato com o suporte, verifique os pontos abaixo, pois eles podem explicar por que a reserva não integrou automaticamente:

**Verificação 1: Categoria, Canal ou Tarifa Recém-Criada**

A reserva entrou para uma categoria, canal ou tarifa que foi **recém-criada** no sistema?

**Se sim, responda:** Foi solicitado o **mapeamento ao suporte** antes da criação da reserva?

**Por que isso é importante:** Quando você cria uma categoria, canal ou tarifa, é necessário que o suporte faça o mapeamento no sistema antes que as reservas possam integrar automaticamente. Se o mapeamento não foi feito, a reserva não vai integrar.

**Solução:** Se você criou uma categoria/canal/tarifa e não solicitou o mapeamento, entre em contato com o suporte para fazer o mapeamento e depois solicitar a importação manual da reserva.

**Verificação 2: Data de Criação da Reserva**

A reserva foi criada **antes** ou **depois** do canal estar mapeado com o HUNIT?

**Importante saber:** Reservas criadas **antes da integração do canal ou categoria** não conseguem ser importadas automaticamente.

**Por que isso acontece:** O sistema só integra automaticamente as reservas criadas após o mapeamento estar ativo. Reservas antigas precisam ser importadas manualmente.

**Solução:** Se a reserva foi criada antes do mapeamento, solicite ao suporte a importação manual da reserva.

**3. Dados Necessários Para Análise**

Se você já verificou os pontos acima e a reserva ainda não apareceu no sistema, será necessário entrar em contato com o suporte.

Para que a equipe possa analisar o problema, envie as seguintes informações:

**Informação 1: Propriedade**

Nome completo do hotel ou pousada.

**Exemplo:** Hotel Mar Azul

**Informação 2: Canal de Origem da Reserva**

Nome do canal onde a reserva foi criada.

**Exemplos:**

* Booking.com
* Expedia
* Decolar
* Hbook
* Airbnb
* Outros canais

**Informação 3: Número da Reserva**

Código de identificação da reserva fornecido pelo canal.

**Importante:** cada canal tem seu próprio formato de código de reserva. Copie exatamente como aparece no canal.

**Exemplos:**

* Booking: 3456789012
* Airbnb: HMABCDEF123
* Expedia: 123456789

**Informação 4: Nome do Hóspede Principal**

Nome completo do hóspede que fez a reserva.

**Exemplo:** Maria da Silva Santos

**Informação 5: Data de Check-in**

Data prevista para entrada do hóspede.

**Formato recomendado:** DD/MM/AAAA

**Exemplo:** 25/12/2024

**5. O Que o Suporte Fará Com Essas Informações**

Com os dados fornecidos, a equipe de suporte irá:

**Etapa 1: Verificar Envio pelo Canal**

Confirmar se a reserva foi enviada pelo canal para o HUNIT.

Às vezes o problema está no canal, que não enviou a informação.

**Etapa 2: Identificar Erros de Integração**

Verificar se houve erro de integração ou falha de comunicação entre o canal e o HUNIT.

Os logs do sistema mostram todos os erros que ocorreram.

**Etapa 3: Validar Mapeamentos**

Conferir se os mapeamentos do canal estão corretos (categorias e tarifas).

Se o mapeamento estiver incorreto ou incompleto, a reserva não integra.

**Etapa 4: Orientar Sobre Ajustes**

Se for necessário fazer algum ajuste (como criar mapeamento ou corrigir configuração), a equipe vai orientar sobre o que precisa ser feito.

**Etapa 5: Importar Manualmente**

Se a reserva não puder integrar automaticamente, a equipe fará a importação manual para o sistema.

**6. Retorno da análise**

Após enviar as informações, o suporte irá:

* Analisar o caso
* Identificar a causa do problema
* Retornar com o status e a solução por e-mail.

## **Glossário de Termos**

|  |  |
| --- | --- |
| **Termo** | **Definição** |
| **HUNIT** | Sistema de channel manager da Hsystem que gerencia a distribuição de quartos e tarifas para diversos canais de venda. |
| **Channel Manager** | Sistema que conecta o hotel aos canais de venda online, sincronizando disponibilidade, preços e reservas. |
| **Integração** | Processo automático de transferência de dados (reservas, disponibilidade, preços) entre o canal e o HUNIT. |
| **Reserva Não Integrada** | Reserva que foi criada no canal mas não apareceu automaticamente no HUNIT. |
| **Canal** | Site de venda online como Booking, Expedia, Airbnb, Decolar, onde os quartos são oferecidos para reserva. |
| **Mapeamento** | Configuração que vincula as categorias e tarifas do hotel com as categorias e tarifas dos canais de venda. |
| **Categoria** | Tipo de quarto ou acomodação (por exemplo: Quarto Standard, Suíte Master, Apartamento Duplo). |
| **Tarifa** | Plano de preço com condições específicas (reembolsável, não reembolsável, com café, sem café, etc.). |
| **Número da Reserva** | Código único de identificação da reserva gerado pelo canal de vendas. |
| **Check-in** | Data de entrada do hóspede no hotel. |
| **Hóspede Principal** | Pessoa em cujo nome a reserva foi feita. |
| **Importação Manual** | Processo realizado pelo suporte para adicionar manualmente uma reserva no sistema quando ela não integrou automaticamente. |
| **Erro de Integração** | Falha na comunicação entre o canal e o HUNIT que impede a transferência automática de dados. |
| **Logs do Sistema** | Registros automáticos de todas as operações e erros que ocorrem no sistema. |
| **Mapeamento de Canal** | Configuração inicial que conecta um canal de vendas ao HUNIT, definindo como as informações serão sincronizadas. |
