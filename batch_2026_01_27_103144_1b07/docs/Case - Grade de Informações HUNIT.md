**Grade de Informações (Gerenciamento de Disponibilidade e Tarifas)**

**HUNIT – Channel Manager**

**1. O Que é a Grade de Informações?**

A Grade de Informações é a tela principal do sistema HUNIT para visualizar e conferir as informações de preços, disponibilidade e restrições que são enviadas para os canais de venda online (OTAs como Booking, Expedia, etc.).

Essa tela funciona como um painel de conferência. Ela não serve para lançar dados novos, mas sim para confirmar se as informações foram enviadas corretamente e consultar o histórico de atualizações.

**2. Para Que Serve?**

A Grade de Informações serve para:

* Conferir os preços enviados para cada canal
* Verificar a disponibilidade de quartos por data
* Identificar restrições aplicadas (fecho de chegada, fecho de saída, stop de venda, estadia mínima)
* Entender por que um canal não está vendendo em determinada data
* Consultar o histórico completo de alterações

**Resumindo:** se você tem dúvidas sobre vendas, bloqueios ou valores em algum canal, a resposta está na Grade de Informações.

**3. Como Acessar a Tela**

Para acessar a Grade de Informações, siga este caminho no sistema:

**HUNIT > CALENDÁRIOS > GRADE DE INFORMAÇÕES**

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_103144_1b07/images/Case_-_Grade_de_Info_image5_0ebb1142.png)

Quando você entrar nessa tela, o sistema mostrará a grade dos canais que estão conectados à sua propriedade.

**4. Como Fazer a Busca (Passo a Passo)**

Antes de visualizar os dados, você precisa fazer a busca seguindo três passos simples:

**Passo 1: Selecionar a Data**

No topo da tela, use o campo de calendário para escolher a data ou período que você quer verificar.

* Você pode navegar diretamente até o dia específico
* A grade será carregada com base na data escolhida

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_103144_1b07/images/Case_-_Grade_de_Info_image6_5ecfd65b.png)

**Importante:** a grade sempre mostra informações relacionadas à data que você selecionou.

**Passo 2: Escolher o Canal**

Depois de selecionar a data, você precisa escolher qual canal quer analisar. Você pode fazer isso de duas formas:

* Rolando a tela manualmente até encontrar o canal desejado
* Usando o seletor de canais no canto superior esquerdo da tela

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_103144_1b07/images/Case_-_Grade_de_Info_image7_991eda9a.png)

Quando você seleciona um canal no seletor, o sistema automaticamente rola a tela até aquele canal.

**Passo 3: Carregar a Grade**

Depois de selecionar a data e o canal, clique em um dos botões de carregamento:

* Botão "Carregar grade" (localizado diretamente na grade do canal)

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_103144_1b07/images/Case_-_Grade_de_Info_image8_88daf707.png)

* Botão "Carregar todos os canais" (localizado acima do calendário)

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_103144_1b07/images/Case_-_Grade_de_Info_image9_e7dfe595.png)

Só após esse carregamento é que as informações de preços, disponibilidade e restrições serão exibidas na tela.

**5. Entendendo as Informações da Grade**

**Preços e Disponibilidade**

Cada célula da grade representa uma data específica. Nela você verá o preço enviado ao canal e a quantidade de quartos disponíveis.

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_103144_1b07/images/Case_-_Grade_de_Info_imagea_b5fec16c.png)

**Atenção**: quando existe stop de venda ou a disponibilidade está zerada (0), o preço aparece com fundo vermelho. Isso indica que as vendas estão fechadas para aquele dia.

**Restrições**

A grade também mostra se existem restrições aplicadas em cada data. Veja os indicadores:

* **Letra C abaixo do preço** = fecho de chegada (check-in)

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_103144_1b07/images/Case_-_Grade_de_Info_imageb_0ac6c565.png)

* **Letra S abaixo do preço** = fecho de saída (checkout)

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_103144_1b07/images/Case_-_Grade_de_Info_imagec_adab19fe.png)

* **Triângulo amarelo no canto do preço** = alerta de que existe alguma regra ativa

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_103144_1b07/images/Case_-_Grade_de_Info_imaged_90d161e6.png)

Para exibir ou esconder as restrições, procure o botão (Esconder restrições ou exibir restrições) na linha azul onde aparece o nome do canal.

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_103144_1b07/images/Case_-_Grade_de_Info_imagee_36b5e273.png)

**6. Como Ver o Histórico de Atualizações**

Se você quer entender quando e como uma informação foi enviada, basta clicar diretamente sobre o valor exibido na grade.

Será aberta a tela de Histórico de Atualizações, mostrando todas as alterações já feitas para aquela data, categoria e canal específicos.

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_103144_1b07/images/Case_-_Grade_de_Info_imagef_c28c3943.png)

**7. Interpretando o Histórico**

O histórico é apresentado em forma de lista, onde:

* As informações mais recentes ficam no topo
* As mais antigas ficam abaixo

A última atualização (linha superior) é a que está recente na Grade de Informações e no canal. Assim, a informação mais recente de cada coluna é a que passa a valer, mesmo que outras colunas tenham sido atualizadas em momentos diferentes.

**Exemplo:**

Marcado em azul são as colunas e em vermelho, as últimas atualizações, que são as que estão refletindo na grade e canal.

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_103144_1b07/images/Case_-_Grade_de_Info_image10_289985fe.png)

**Colunas do Histórico**

* **Data de alteração:** dia e hora em que a atualização foi enviada para o canal
* **Processamento:** data e hora em que o canal recebeu e aceitou a informação
* **Tarifa de venda:** valor de preço atualizado
* **Disponibilidade:** quantidade de quartos disponíveis enviada
* **Estadia mínima:** número mínimo de noites exigidas
* **Chegada:** indica se o check-in está aberto ou fechado
* **Saída:** indica se o checkout está aberto ou fechado
* **Venda:** indica se as vendas estão abertas ou se existe stop
* **Usuário:** quem realizou o lançamento
* **Origem:** por onde a atualização foi feita
* **Pacote ID:** número gerado pela atualização; ao clicar, você visualiza exatamente o que foi enviado
* **Descrição:** campo com informações adicionais

**8. Conclusão**

A Grade de Informações é a ferramenta mais importante para controle e conferência das suas vendas online. Com ela você pode:

* Verificar se um canal está vendendo
* Identificar restrições que estão bloqueando vendas
* Confirmar se o preço correto foi enviado
* Saber quem fez cada atualização

**Sempre que surgir dúvida sobre preços, disponibilidade ou restrições, consulte a Grade de Informações antes de qualquer outra ação.**

|  |  |
| --- | --- |
| **Termo** | **Definição** |
| **Canal** | São os sites de venda online (como Booking, Expedia, Decolar) onde seus quartos são oferecidos para reserva. |
| **OTA (Online Travel Agency)** | Agências de viagem online. É outro nome para os canais de venda. |
| **PMS (Property Management System)** | Sistema de gerenciamento do hotel. É o programa que controla reservas, check-in, checkout e outras operações do dia a dia. |
| **Disponibilidade** | Quantidade de quartos livres que estão sendo oferecidos para venda em cada data. |
| **Tarifa de Venda** | O preço do quarto que está sendo enviado para os canais de venda. |
| **Restrições** | Regras aplicadas a determinadas datas, como fecho de chegada, fecho de saída ou estadia mínima. |
| **Fecho de Chegada (Check-in)** | Quando ativado, não é permitido fazer check-in naquela data específica. O hóspede não pode entrar no hotel naquele dia. |
| **Fecho de Saída (Checkout)** | Quando ativado, não é permitido fazer checkout naquela data específica. O hóspede não pode sair do hotel naquele dia. |
| **Stop de Venda (Fecho de Vendas)** | Quando ativado, as vendas estão completamente bloqueadas para aquela data. Ninguém consegue fazer reserva. |
| **Estadia Mínima** | Número mínimo de noites que o hóspede precisa reservar. Por exemplo, se a estadia mínima é 3, o hóspede precisa ficar pelo menos 3 noites. |
| **Histórico de Atualizações** | Registro de todas as alterações feitas em preços, disponibilidade e restrições. Mostra quem fez cada mudança, quando foi feita e quando o canal recebeu a informação. |
| **True / False** | **True** significa fechado no sistema. **False** significa aberto no sistema. |
| **HUNIT -- INTEGRATION** | Indica que a atualização foi enviada automaticamente por um sistema integrado (como o PMS), e não manualmente por um usuário. |
| **Pacote ID** | Número de identificação gerado quando uma atualização é processada. Ao clicar nele, você pode ver todos os detalhes do que foi enviado. |
