**Meios de Pagamento**

**Hbook (Motor de Reservas)**

**1. Aba Meios de Pagamento:**

* A tela de Meios de Pagamento permite configurar as formas de pagamento que o hotel disponibilizará aos hóspedes no momento da compra.
* No menu principal, clique em “Propriedade”, depois em “Geral” e selecione a aba “Meios de Pagamento”.

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_144114_87c5/images/Aba_-_Meios_de_Pagam_imageb_95714ffc.png)

**2**. **O administrador pode selecionar entre diferentes opções, como:**

* **Pagamento com cartão de crédito:** possibilita que o hóspede pague de forma prática e segura utilizando cartão.
* Ao disponibilizar essa forma de pagamento, o hotel pode escolher quais bandeiras ele aceita e se deseja incluir o valor dos extras nas cobranças via cartão de crédito.

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_144114_87c5/images/Aba_-_Meios_de_Pagam_imaged_1005cfe7.png)

**3.Você pode trabalhar de duas formas:**

* Manual: o hóspede informa os dados e você realiza a transação na maquininha
* Integrado: o pagamento é feito automaticamente, com integração via Cielo, Rede, Stone ou GetNet (via integrações homologadas pela HSystem, confira na nossa base de conhecimento as adquirentes compatíveis)

**4.Para habilitar a transação automática:**

* Contrate o Módulo E-commerce com sua adquirente
* Contrate o Módulo HPay com a HSystem

**5.Pagamento por depósito:**

* Opção em que o cliente realiza a transferência via depósito bancário.
* Ao disponibilizar a forma de pagamento por depósito, o hotel precisa preencher corretamente as informações bancárias, como: nome do banco, número da conta, agência, CPF/CNPJ e favorecido. Também tem um campo de texto descritivo, que o hotel pode colocar uma mensagem ao hospede. Essas informações serão utilizadas pelo hóspede para realizar a transação.

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_144114_87c5/images/Aba_-_Meios_de_Pagam_image_302abb20.png)

* Existe a opção **“Ocultar todos os dados para depósito no motor de reserva”**:
* Se **desmarcada**, os dados bancários serão exibidos na tela “**Finalizando sua reserva**”, antes do cliente concluir a reserva.

* No campo **“Habilitar depósito apenas para reservas que antecedem o check-in em pelo menos a seguinte quantidade de dias úteis”**, o hotel pode definir uma regra de prazo mínimo para aceitar depósitos. Caso não preencha nada, a forma de pagamento por deposito será apresentada em estadias para o mesmo dia do check-in (exemplo: hoje é dia 27/09/25 e estou buscando reserva com entrada dia 27/09/25).

* **Considera apenas dias úteis**, posso realizar busca de reserva no sábado, mas para meu check-in tem que haver X dias uteis para reservar como depósito.

* **Pagamento por PIX**: método instantâneo de pagamento, facilitando a confirmação rápida da reserva.
* Ao disponibilizar essa forma de pagamento, o hotel precisa preencher corretamente os seguintes campos:
* **Tipo de Chave PIX**: selecionar se será telefone, e-mail, CPF/CNPJ ou chave aleatória (EVP).
* **Chave PIX**: informar o dado correspondente, sem caracteres especiais (incluir código do país e DDD).
* **Nome do Titular da Conta**: exatamente como registrado no banco, sem caracteres especiais.
* **Cidade do Titular da Conta:** local vinculado à conta bancária, sem caracteres especiais.

**7.Além disso, o hotel pode optar por:**

* **Incluir valor dos extras nas cobranças via PIX**: se habilitado, valores adicionais serão somados automaticamente.
* **Configurar antecedência mínima para pagamento**: definir o número de dias úteis antes do check-in em que o PIX será aceito. Caso esse campo não seja preenchido, o sistema permitirá pagamentos inclusive para reservas com check-in no mesmo dia.

Há também campos de **descrição no pagamento** e **texto complementar**, que aparecem como instruções para o cliente, ajudando a orientar o envio do comprovante e os prazos para confirmação.

**8.Você pode trabalhar de duas formas:**

* Manual: O hotel coloca na descrição orientações para o hospede de prazo de pagamento e como enviar o comprovante, caso o hospede não encaminhe dentro do prazo o comprovante de pagamento o hotel, de forma manual, realiza o cancelamento da reserva.
* Integrado: o pagamento pix é feito automaticamente, com integração via Cielo ou Bradesco. (via integrações homologadas pela HSystem, confira na nossa base de conhecimento as adquirentes compatíveis)

**9.Para habilitar a transação automática:**

* Contrate o Módulo E-commerce com sua adquirente ou banco.
* Contrate o Módulo HPix com a HSystem
* **Pagamento direto no hotel:** o hóspede efetua o pagamento somente no momento do check-in ou checkout.
* Quando o hotel opta por aceitar a forma de pagamento direto no hotel, é necessário configurar o campo “Permitido iniciando (X) horas antes do Check-in” para que essa opção seja apresentada ao hóspede no momento da reserva.
* A configuração deve ser feita em horas.
* Caso o hotel queira permitir essa forma de pagamento durante todo o ano, recomenda-se preencher o campo com 9999999 horas.
* Importante: em reservas não reembolsáveis, a opção de pagamento direto no hotel não será exibida ao hóspede.
* Há também um campo de texto complementar, que aparece como instruções para o cliente, ajudando a orientar como funciona o pagamento no hotel.

**Glossário**

|  |  |
| --- | --- |
| **Termo** | **Significado** |
| Aba | Seção ou página dentro de um sistema. Como as abas de uma pasta ou arquivo. |
| Adquirente | Empresa que processa pagamentos com cartão de crédito. Exemplos: Cielo, Rede, Stone, GetNet. |
| Agência | Número que identifica a agência bancária onde a conta está registrada. |
| Bandeira | Empresa responsável pela rede de processamento de cartões. Exemplos: Visa, Mastercard, Elo, American Express. |
| Cartão de Crédito | Forma de pagamento que permite comprar agora e pagar depois, geralmente com possibilidade de parcelamento. |
| Chave PIX | Identificador único usado para receber pagamentos via PIX. Pode ser telefone, e-mail, CPF/CNPJ ou chave aleatória. |
| Dias Úteis | Dias da semana excluindo sábados, domingos e feriados. Normalmente de segunda a sexta-feira. |
| E-commerce | Comércio eletrônico. Sistema que permite vender e receber pagamentos pela internet. |
| EVP | Chave PIX aleatória gerada automaticamente pelo banco. Sequência de letras e números. |
| Extras | Serviços adicionais à hospedagem. Exemplos: café da manhã, traslado, frigobar, late check-out. |
| Favorecido | Pessoa ou empresa que receberá o pagamento. Nome do titular da conta bancária. |
| Gateway de Pagamento | Sistema intermediário que processa pagamentos online de forma segura. |
| HBook | Nome do sistema de gerenciamento de reservas usado pelo hotel. |
| Hóspede | Pessoa que se hospeda ou faz reserva no hotel. Cliente do hotel. |
| HPay / HPix | Módulos da HSystem que permitem processar pagamentos automaticamente. |
| Integrado/Automático | Quando o sistema processa o pagamento automaticamente, sem intervenção manual. |
| Manual | Quando o hotel precisa fazer o processo manualmente (passar cartão na maquininha, conferir comprovante, etc.). |
| Maquininha | Equipamento usado para processar pagamentos com cartão de crédito ou débito. |
| Meio de Pagamento | Forma como o cliente pode pagar. Exemplos: cartão, PIX, dinheiro, depósito. |
| Não Reembolsável | Tipo de tarifa que não permite cancelamento com devolução do dinheiro. |
| PIX | Sistema de pagamento instantâneo brasileiro. Permite transferir dinheiro em segundos, 24 horas por dia. |
| PIX Cópia e Cola | Código de texto que pode ser copiado e colado no aplicativo do banco para fazer pagamento via PIX. |
| QR Code | Código de barras quadrado que pode ser escaneado pela câmera do celular para fazer pagamento. |
| Reserva | Solicitação de hospedagem feita com antecedência. Garantia de vaga no hotel para determinadas datas. |
| Tarifa | Tipo ou categoria de preço para hospedagem. Exemplos: Tarifa Promocional, Tarifa de Grupo, Tarifa Não Reembolsável. |
| Transação | Operação de pagamento. Transferência de dinheiro de uma conta para outra. |
