# **Atualização de Preço e Restrições – Correção, Limitações e Replicações**

## **1. Sobre correção e limitação de atualizações preço e restrições**

Ao trabalhar com atualização de **preços** e **restrições** (como estadia mínima, fechamentos de chegada e saída), é importante entender como funcionam as correções e quais são as limitações do sistema.

Este guia esclarece as dúvidas mais comuns sobre:

* Como corrigir informações incorretas
* Limitações de cópia entre anos
* Limitações de cópia entre categorias
* Quando é possível replicar valores

## **2. Como Corrigir Preço ou Restrição Incorreta**

### **Pergunta: Lancei preço ou restrição incorreta. Como desfazer?**

Atualmente, **não é possível "desfazer" uma atualização** já enviada ao canal.

### **Como Corrigir**

A **única forma de corrigir** um preço ou restrição incorreta é:

**Passo 1: Realizar Nova Atualização**

Faça uma nova atualização no sistema.

**Passo 2: Informar Dados Corretos**

Informe as informações corretas que você deseja enviar.

**Passo 3: Enviar ao Canal**

Envie a atualização. Ela vai **sobrepor** a informação anterior.

### **Como Funciona**

O sistema sempre considera a **última atualização enviada e aceita pelo canal**.

Isso significa que:

* A atualização mais recente substitui a anterior
* Não há histórico de "versões" que você possa restaurar
* Cada nova atualização sobrescreve completamente a anterior

### **Exemplo Prático**

**Situação:** Você lançou R$ 250,00 mas o correto era R$ 350,00.

**Solução:**

1. Acesse o sistema de atualizações
2. Selecione a mesma data e categoria
3. Lance o valor correto: R$ 350,00
4. Envie a atualização
5. O valor de R$ 250,00 será substituído por R$ 350,00

## **3. Copiar Valores de Um Ano Para Outro**

A gestão tarifária é uma das etapas mais importantes da distribuição hoteleira, pois envolve valores, regras de venda e estratégias que impactam diretamente a ocupação e a rentabilidade do hotel. Pensando nisso, é comum surgir a dúvida sobre a possibilidade de **copiar o tarifário de um ano para outro**, visando ganhar tempo no processo de configuração.

Neste artigo, explicamos como esse processo funciona, por que essa funcionalidade não está disponível no sistema e qual é a forma correta de realizar as configurações para um novo período, garantindo segurança nas informações enviadas aos canais de venda.

### **Pergunta: É possível copiar valores ou restrições de um ano para outro?**

**Não é possível copiar valores e/ou restrições de um ano para outro.**

### **Por Que Não É Possível**

Isso acontece porque:

**Motivo 1: Dias da Semana Mudam**

Os dias da semana **não se repetem** de um ano para outro.

**Exemplo:**

* 25 de dezembro de 2024 = quarta-feira
* 25 de dezembro de 2025 = quinta-feira

**Motivo 2: Finais de Semana Mudam**

Finais de semana, feriados e datas comemorativas caem em dias diferentes a cada ano.

**Motivo 3: Precificação Por Dia da Semana**

Muitos hotéis trabalham com valores diferentes para:

* Dias úteis (segunda a quinta)
* Finais de semana (sexta, sábado, domingo)
* Períodos especiais (férias, feriados)

### **Problemas Que Seriam Causados**

Copiar dados de um ano para outro poderia gerar **impactos negativos**, como:

* Valores incorretos em fins de semana
* Restrições incompatíveis com o novo calendário
* Bloqueio de vendas em datas indevidas
* Perda de receita por precificação inadequada

### **Solução**

**A atualização deve ser feita manualmente**, respeitando o novo calendário do ano seguinte.

**Como fazer:**

* Analise o calendário do novo ano
* Identifique quais dias são finais de semana, feriados etc.
* Lance os valores adequados para cada dia da semana
* Configure as restrições conforme a estratégia do novo ano

## **4. Copiar Valores de Uma Categoria Para Outra**

### **Pergunta: É possível copiar valores de uma categoria para outra?**

**Não é possível copiar valores de uma categoria para outra diferente.**

**Exemplo do que NÃO é possível:** Copiar valores da **categoria Luxo** para **categoria Standard**.

### **Por Que Não É Possível**

Cada categoria possui:

**Característica 1: Propriedades Próprias**

Cada categoria tem características únicas (tamanho, vista, comodidades, etc.).

**Característica 2: Capacidades Diferentes**

Categorias têm capacidades diferentes de hóspedes.

**Característica 3: Preços Diferentes**

Categorias têm preços diferentes por serem superiores ou inferiores.

**Característica 4: Estratégias Distintas**

Cada categoria pode ter estratégia de venda diferente.

## **5. Quando É Possível Copiar Valores**

A replicação de valores é permitida **somente** nas seguintes condições:

### **Condição 1: Mesmo Canal Para Outro Canal**

Copiar valores **de um canal para outro**.

### **Condição 2: Mesma Categoria**

**Mantendo a mesma categoria** em ambos os canais.

### **Condição 3: Categoria Existe em Ambos**

Desde que:

* A categoria exista em ambos os canais
* Esteja corretamente mapeada em ambos

### **Exemplo Permitido**

**Copiar valores da categoria Luxo do Booking para a categoria Luxo da Expedia**

**Por que é permitido:**

* É a mesma categoria (Luxo)
* Vai de um canal (Booking) para outro canal (Expedia)
* A categoria existe e está mapeada em ambos os canais

## **6. Como Fazer Replicação Permitida**

Se você quer copiar valores de um canal para outro (mesma categoria):

### **Passo 1: Verificar Se a Categoria Existe**

Confirme que a categoria está criada e mapeada em ambos os canais.

**Se não estiver e deseja ter:** Realize a criação no canal e depois solicite o mapeamento da categoria com o nosso suporte.

### **Passo 2: Solicitação ao suporte**

Apos essa verificação, entre em contato com o suporte para solicitar a replicação.

## **Glossário de Termos**

|  |  |
| --- | --- |
| **Termo** | **Definição** |
| **Atualização** | Envio de novos valores de preços ou restrições para os canais de venda. |
| **Restrição** | Regras aplicadas às vendas, como estadia mínima, fecho de chegada/saída, stop de venda. |
| **Sobrepor** | Substituir uma informação anterior por uma nova. |
| **Replicação** | Copiar valores e restrições de um canal para outro. |
| **Categoria** | Tipo de acomodação (Quarto Standard, Suíte Luxo, etc.). |
| **Canal** | Site de vendas online como Booking, Expedia, Airbnb. |
| **Mapeamento** | Conexão entre a categoria do canal e o sistema HUNIT/PMS. |
| **Estadia Mínima** | Número mínimo de noites que o hóspede precisa reservar. |
| **Fecho de Chegada** | Bloqueio de check-in em uma data específica. |
| **Fecho de Saída** | Bloqueio de checkout em uma data específica. |
| **Stop de Venda** | Bloqueio total de vendas para uma data ou período. |
| **Precificação** | Estratégia de definição de preços por período, dia da semana, etc. |
| **Validação** | Verificação de que as informações foram atualizadas corretamente. |
| **Calendário Anual** | Planejamento de preços e restrições para o ano inteiro. |
| **Última Atualização** | A atualização mais recente enviada e aceita pelo canal, que está atualmente ativa. |
