**Guia do Voucher de Reservas**

**HBOOK**

**O que é um voucher?**

O Voucher é o documento que confirma a reserva. É como comprovante que mostra todos os detalhes: Quem reservou, qual quarto, quanto pagou e outras informações importantes.

**Por que é importante?**

O voucher serve como prova da reserva tanto para o hóspede quanto para o hotel. Ele registra todos os acordos feitos no momento da compra.

**1.Como acessar o voucher**

**HBOOK>**

1. Clique em "RESERVAS" no menu superior
2. Clique em "LISTA DE RESERVAS"

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_20_154550_09d9/images/HBOOK_-_Como_Visuali_image2_c186ca8a.png)

**2.Informações Básicas (no topo)**

* **RESERVA Nº** - Código de confirmação ou número da reserva;
* **Canal de origem** - Como a reserva é dentro do motor, sempre será HBOOK;
* **Quantidade de hóspede** - Para quantas pessoas foi reservado;
* **Status** - Qual status se encontra a reserva (Ativa, Confirmada, Canceladas, NoShow, Abandonadas, Erro no pagamento, Orçamentos e Recuperadas);
* **Dispositivo** - Por qual dispositivo foi realizado a reserva (Desktop ou Mobile(celular).

O voucher completo será exibido na tela.

**12. Políticas e condições:**

* Registro e condições que o cliente concordou durante a compra (li e concordo/aceite), caso precise verificar as informações que constavam quando a reserva foi realizada, qual a versão que o cliente recebeu quando reservou.
* **Políticas** aceitas pelo hóspede.
* Em caso de contestação por parte do hóspede sobre as políticas aceitas no momento da reserva, o hotel pode verificar diretamente no **voucher da reserva** as informações que o hóspede **aceitou no ato da compra**. Esse documento registra as condições confirmadas no momento da finalização da reserva, servindo como comprovante do aceite das políticas vigentes.

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_20_154550_09d9/images/HBOOK_-_Como_Visuali_image3_1ba679cb.png)

**13. Ações possíveis pelo hotel na Reserva**

* **Imprimir o voucher**
* **Reenviar o e-mail ao hóspede;**
* **Reenviar o e-mail que foi recebido pelo hóspede para o hotel;**
* **reenviar o e-mail que foi recebido pelo hóspede para outro e-mail;**
* **Download do retorno de pagamento, caso de falha ou sucesso na transação (Hpay);**
* **Editar reserva (Somente tem como editar E-mail e telefone na reserva);**
* **Histórico de integração (envio para o channel, status do envio e XML);**
* **Marcar NoShow (Hóspede não compareceu);**
* **Cancelar reserva.**

**Glossário**

|  |  |
| --- | --- |
| **Termo** | **Significado** |
| **Canal** | Origem da reserva. No HBOOK sempre será "HBOOK" (site do hotel). |
| **CPF** | Cadastro de Pessoa Física. Documento de identificação brasileiro. |
| **Dispositivo** | Equipamento usado. Pode ser Desktop (computador) ou Mobile (celular). |
| **Extras** | Serviços adicionais. Exemplos: café da manhã, transfer, massagem. |
| **HBOOK** | Motor de Reservas. Sistema de reservas diretas do hotel. |
| **HPAY** | Sistema de pagamento. Processa cartões de crédito online. |
| **HUNIT** | Channel Manager. Gerencia canais de venda (Booking, Expedia, etc.). |
| **IP** | Endereço do computador na internet. Identifica de onde veio a reserva. |
| **ISS** | Imposto Sobre Serviços. Taxa municipal cobrada sobre serviços. |
| **No-Show** | Não comparecimento. Quando o hóspede reservou, mas não apareceu. |
| **Passaporte** | Documento de identificação internacional. Para hóspedes estrangeiros. |
| **PMS** | Sistema de Gerenciamento do Hotel. Sistema interno de operação. |
| **Pré-pagamento** | Valor pago antes da chegada. Pagamento antecipado. |
| **Status** | Situação da reserva. Exemplos: ativa, cancelada, confirmada. |
| **Tarifa** | Plano de preço escolhido. Exemplos: standard, promocional, não reembolsável. |
| **Taxa** | Cobrança adicional. Exemplos: limpeza, turismo, ISS. |
| **Voucher** | Comprovante da reserva. Documento com todos os detalhes. |
