# **Grade de Disponibilidade – Calendário Hunit**

## **O que é a Grade de Disponibilidade**

A **Grade de Disponibilidade** é uma tela localizada dentro do módulo **Calendário** do sistema **Hunit**. Ela é responsável por exibir e controlar a **quantidade de quartos disponíveis para venda**, informação essencial para que os canais de venda saibam se podem ou não receber novas reservas.

Essa tela pode ter dois comportamentos diferentes, de acordo com o perfil do hotel:

* Hotéis **com PMS conectado**
* Hotéis **sem PMS conectado**

Em ambos os casos, a Grade de Disponibilidade tem a função de garantir que os canais de venda recebam informações corretas e atualizadas, evitando erros e overbooking.

## **Onde localizar a Grade de Disponibilidade no sistema**

Para acessar essa tela no Hunit, siga o caminho abaixo:

**Menu superior → Calendário → Grade de Disponibilidade**

Ao acessar essa opção, o sistema exibirá um calendário com as disponibilidades organizadas por data.

## **Para que serve a Grade de Disponibilidade**

A Grade de Disponibilidade serve para **acompanhar e controlar o inventário de quartos** que está sendo compartilhado com os canais de venda, como Booking, Expedia e outros.

Ela permite que o hoteleiro:

* Verifique quantos quartos estão disponíveis em cada data
* Confirme se as informações estão corretas nos canais
* Garanta alinhamento entre sistema, PMS e canais de venda

A forma de utilização varia conforme a existência ou não de integração com PMS.

## **Uso da Grade de Disponibilidade para clientes COM PMS conectado**

Quando o hotel possui um **PMS (Sistema de Gestão da Propriedade)** integrado ao Hunit, a Grade de Disponibilidade tem função **exclusivamente informativa**.

### **Como funciona nesse cenário**

* A disponibilidade é enviada automaticamente do PMS para o Hunit
* O Hunit repassa essas informações aos canais de venda conectados
* A Grade de Disponibilidade exibe exatamente o que está sendo recebido e compartilhado

Ou seja, essa tela funciona como um **painel de conferência**, permitindo que o cliente valide se os dados enviados pelo PMS estão corretos.

### **O que o cliente pode fazer**

Nesse caso, o cliente pode:

* Conferir as quantidades recebidas do PMS
* Acompanhar se os canais estão recebendo as informações corretamente
* Identificar possíveis divergências

Importante: Quando existe PMS conectado, **não é possível editar manualmente a disponibilidade no Hunit**, pois a informação sempre deve vir do PMS.

## **Uso da Grade de Disponibilidade para clientes SEM PMS conectado**

Para hotéis que **não possuem PMS integrado**, a Grade de Disponibilidade tem um papel ainda mais importante.

Nesse cenário, além de visualizar as informações, o cliente também pode **atualizar manualmente a disponibilidade** e enviá-la para os canais de venda.

### **Como funciona nesse cenário**

* O Hunit se torna a ferramenta principal de controle de inventário
* As disponibilidades são definidas diretamente no sistema
* As informações atualizadas são enviadas aos canais conectados

## **Como atualizar a disponibilidade manualmente (sem PMS)**

Na Grade de Disponibilidade ou na tela de atualizações:

1. Clique sobre o número da disponibilidade da data desejada
2. Digite a nova quantidade de quartos disponíveis
3. O campo ficará destacado, indicando que a alteração está pendente
4. Após revisar os valores, clique no botão **Atualizar**

Somente após clicar em **Atualizar** a nova disponibilidade será enviada aos canais.

## **Entendendo a visualização da Grade de Disponibilidade**

Na tela da grade:

* Cada **coluna** representa uma data
* Cada **linha** representa uma categoria de quarto
* Os números exibidos correspondem à quantidade de quartos disponíveis

Essa visualização facilita o acompanhamento diário do inventário.

## **Importância da Grade de Disponibilidade**

Manter a Grade de Disponibilidade sempre correta é fundamental para:

* Evitar overbooking
* Garantir que os canais vendam apenas o que está disponível
* Manter a operação organizada
* Evitar perdas financeiras e problemas com hóspedes

Essa tela deve ser conferida com frequência, principalmente em períodos de alta ocupação.

## **Glossário**

|  |  |
| --- | --- |
| **Termo** | **Descrição** |
| **Calendário** | Módulo do Hunit onde são gerenciadas tarifas, disponibilidades e restrições. |
| **Disponibilidade** | Quantidade de quartos disponíveis para venda. |
| **Canal de venda** | Plataforma onde as diárias são comercializadas (Booking, Expedia, entre outros). |
| **Channel Manager** | Sistema responsável por distribuir as informações do hotel para os canais de venda. |
| **PMS (Property Management System)** | Sistema de gestão do hotel, responsável pelo controle operacional, reservas, hóspedes e inventário. |
| **Overbooking** | Situação em que são vendidas mais unidades do que o disponível no hotel. |
