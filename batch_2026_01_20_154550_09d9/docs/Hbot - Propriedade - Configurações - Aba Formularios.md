## **Aba Formulários da Configuração da propriedade – HBOT (HAI)**

### **1. O que é a aba Formulários**

A aba **Formulários** é responsável por definir **quais informações o chatbot irá solicitar ao hóspede** durante interações que envolvem coleta de dados, como:

* Pedidos de cotação;
* Intenções de reserva;
* Identificação inicial do cliente;
* Outros fluxos em que é necessário reconhecer quem está entrando em contato.

É nessa tela que você decide **quais campos aparecem no formulário** e se cada informação será **opcional ou obrigatória**, garantindo equilíbrio entre experiência do cliente e necessidade comercial.

### **2. Objetivo da funcionalidade**

O principal objetivo da aba Formulários é:

* Padronizar a coleta de dados dos hóspedes;
* Garantir que o time receba as informações essenciais para contato e conversão;
* Permitir flexibilidade para adaptar o formulário ao perfil da propriedade;
* Melhorar a qualificação dos leads gerados pelo chatbot.

### **3. Caminho de acesso**

Menu superior → Propriedade → Configuração → Formulários

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_20_154550_09d9/images/Hbot_-_Propriedade_-_imagea_4ec1bb9f.png)

Essa aba é responsável por determinar **quais informações o bot solicitará ao hóspede** durante interações que envolvem coleta de dados — como pedidos de orçamento, intenções de reserva, cadastro inicial e outros fluxos que dependem de identificação do cliente.

É aqui que você controla **quais campos aparecem no formulário** e se cada um deles deve ser **opcional ou obrigatório**.

A tela apresenta três grupos de informações principais que podem ser solicitadas ao hóspede:

* **Nome**
* **E-mail**
* **Telefone / WhatsApp**

Para cada uma delas, você encontrará dois tipos de controles:

1. **Habilitar solicitação** (ex.: *Pedir nome*)
2. **Tornar obrigatório** (ex.: *Nome é obrigatório*)

Essas opções permitem personalizar o formulário de acordo com as necessidades da sua propriedade.

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_20_154550_09d9/images/Hbot_-_Propriedade_-_image2_81f48ee6.png)**4. Pedir Nome**

* Ativando essa opção, o bot passa a solicitar o **nome completo do hóspede** no formulário.

### **Nome é obrigatório**

Se habilitado, o hóspede **só conseguirá enviar o formulário** após preencher o nome.
Caso esteja desativado, o nome será opcional.

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_20_154550_09d9/images/Hbot_-_Propriedade_-_image4_19dc1b53.png)**5. Pedir E-mail**

* Quando ativado, o Hbot exibirá um campo de **e-mail** para o hóspede preencher.

### **6. E-mail é obrigatório**

Se esta opção estiver ligada, o formulário **exigirá um e-mail válido** antes de permitir o envio.
Ideal para propriedades que utilizam confirmação por e-mail.

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_20_154550_09d9/images/Hbot_-_Propriedade_-_image5_3aaa0daf.png)**3. Pedir Telefone / WhatsApp**

Habilitando essa opção, o bot solicitará um número de telefone ou WhatsApp ao hóspede.

### **Telefone/WhatsApp é obrigatório**

* Quando ligado, o hóspede deve informar o telefone para concluir o formulário.
  Recomendado para propriedades que fazem contato comercial principalmente via WhatsApp.

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_20_154550_09d9/images/Hbot_-_Propriedade_-_image8_1b7ef6c1.png)**7. Salvando as Alterações**

* Depois de selecionar os campos desejados, clique no botão **Salvar** para aplicar as configurações.
  A partir desse momento, todos os formulários gerados pelo Hbot seguirão o padrão definido nesta tela.

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_20_154550_09d9/images/Hbot_-_Propriedade_-_image9_e674ac57.png)

## **Glossário**

|  |  |
| --- | --- |
| **Hbot** | Sistema de chatbot e atendimento multicanal da HSystem. |
| **Formulário** | Conjunto de campos apresentados ao hóspede para coleta de informações pessoais durante a interação com o bot. |
| **Lead** | Contato qualificado gerado a partir de uma interação do cliente com o chatbot. |
| **Intenção** | Ação ou objetivo identificado pelo chatbot durante a conversa (ex.: cotação, reserva, dúvidas). |
| **Perfil do Cliente** | Área onde ficam centralizados os dados coletados do hóspede durante o atendimento. |
| **Tags** | Identificadores usados para classificar contatos ou conversas (ex.: cotação, sem disponibilidade). |
