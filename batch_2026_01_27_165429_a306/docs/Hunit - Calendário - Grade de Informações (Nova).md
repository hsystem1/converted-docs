# **Grade de Informações (Nova) – Calendário Hunit**

## **O que é a Grade de Informações (Nova)**

A **Grade de Informações (Nova)** é uma tela localizada dentro do módulo **Calendário** do sistema **Hunit**. Ela foi desenvolvida para permitir que o hoteleiro acompanhe, de forma clara e organizada, **todas as informações que estão sendo enviadas para os canais de venda**, como tarifas, disponibilidade de quartos e restrições.

Essa tela funciona como um painel de conferência e análise. Ou seja, ela mostra exatamente **o que está sendo praticado em cada canal**, ajudando o usuário a validar se as informações estão corretas e alinhadas com a estratégia do hotel.

## **Onde localizar a Grade de Informações (Nova)**

Para acessar essa tela no sistema Hunit, siga o caminho abaixo:

**Menu superior → Calendário → Grade de Informações (Nova)**

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_165429_a306/images/Hunit_-_Calenda%CC%81rio__image8_89a01918.png)

Ao acessar essa opção, o sistema abrirá uma tela de seleção antes de exibir o calendário. Isso acontece porque, nessa nova estrutura, o usuário escolhe **quais canais deseja visualizar**, tornando a navegação mais rápida e objetiva.

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_165429_a306/images/Hunit_-_Calenda%CC%81rio__image9_ea87d579.png)

## **Para que serve a Grade de Informações (Nova)**

A Grade de Informações (Nova) tem três objetivos principais:

### **1. Acompanhar informações enviadas aos canais**

Essa tela permite verificar:

* Tarifas enviadas para cada canal
* Disponibilidade de quartos
* Restrições aplicadas por data

Dessa forma, o usuário consegue confirmar se o que está configurado no sistema realmente está sendo refletido nos canais de venda, como Booking, Expedia, Hotelbeds, entre outros.

### **2. Conferir canais individualmente ou em conjunto**

Um dos grandes diferenciais da Grade de Informações (Nova) é a possibilidade de selecionar:

* Apenas um canal específico, ou
* Vários canais ao mesmo tempo

Isso facilita tanto a análise pontual quanto a visualização geral da operação.

Por exemplo:

* Verificar apenas o Booking para conferir uma alteração recente
* Selecionar vários canais para validar se todos estão com os mesmos valores e disponibilidades

### **3. Fazer análise comparativa entre canais**

A Grade de Informações (Nova) também é uma ferramenta estratégica.

Ao visualizar canais diferentes lado a lado, o hoteleiro consegue:

* Comparar tarifas praticadas em cada canal
* Identificar diferenças de restrições
* Avaliar se a estratégia comercial está sendo aplicada corretamente

Essa análise ajuda na tomada de decisão e evita divergências que podem impactar as vendas.

## **Como selecionar os canais para visualização**

Ao acessar a Grade de Informações (Nova), o sistema exibirá uma lista com todos os canais conectados ao Hunit.

### **Visualizar canais específicos**

* Clique diretamente no nome do canal desejado
* É possível selecionar um ou vários canais

Essa opção é indicada quando o objetivo é analisar ou conferir informações de um canal específico.

### **Visualizar todos os canais**

* Clique no botão **Marcar Todos**

Essa opção é ideal para uma visão completa da distribuição de tarifas e disponibilidades.

Após realizar a seleção, clique no botão verde **Ir para a Grade** para carregar o calendário.

## **Entendendo a estrutura da Grade de Informações**

Depois que a grade é carregada, o usuário visualiza um calendário organizado por datas.

Cada coluna representa um dia, e cada linha representa uma categoria ou tarifa, de acordo com a configuração do hotel.

Na grade, são exibidas as seguintes informações:

* Valores (tarifas)
* Disponibilidade de quartos
* Indicadores de restrições

## **Como interpretar as informações exibidas**

### **Células com fundo branco**

Quando as células aparecem com fundo branco, significa que:

* A venda está aberta
* Existe disponibilidade
* O canal está apto a receber reservas

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_165429_a306/images/Hunit_-_Calenda%CC%81rio__imagea_e56aaf2f.png)

### **Células com fundo vermelho**

O fundo vermelho indica que a venda está fechada para aquela data. Isso pode ocorrer quando:

* A disponibilidade está zerada (valor 0)

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_165429_a306/images/Hunit_-_Calenda%CC%81rio__imageb_765b2431.png)

* A venda foi fechada manualmente (stop de venda). Pode conter disponibilidade, mas está com o fundo vermelho, indicando que foi realizado uma pausa da venda.

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_165429_a306/images/Hunit_-_Calenda%CC%81rio__imagec_a0656710.png)

### **Triângulo amarelo – Indicação de restrições**

O triângulo amarelo sinaliza que existe alguma restrição aplicada naquela data.

Ao clicar em **Exibir Restrições**, o sistema mostrará uma linha adicional com o detalhe da restrição:

* **Número**: Indica a estadia mínima (ex.: 2 = mínimo de 2 noites)
* **C**: Fechamento de chegada (não permite check-in)
* **S**: Fechamento de saída (não permite checkout)

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_165429_a306/images/Hunit_-_Calenda%CC%81rio__imaged_2d60eab8.png)

## **Menu superior da Grade de Informações**

Na parte superior da tela, o usuário encontra algumas ações importantes:

### **Exibir Restrições**

Permite visualizar ou ocultar as restrições aplicadas no calendário.

**Quando usar:** sempre que houver um triângulo amarelo na grade.

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_165429_a306/images/Hunit_-_Calenda%CC%81rio__imagee_c0b8c8cd.png)

### **Imprimir**

Gera um arquivo para impressão exatamente com as informações exibidas na tela.

É útil para conferência, auditoria ou controle interno.

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_165429_a306/images/Hunit_-_Calenda%CC%81rio__imagef_18ef01cc.png)

### **Carregar Grade**

Esse botão é essencial para o uso da tela.

Sempre que o calendário for aberto:

* As informações aparecem inicialmente em branco
* Isso é um comportamento padrão do sistema

Ao clicar em **Carregar Grade**, o Hunit busca e exibe os dados do(s) canal(is) selecionado(s).

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_165429_a306/images/Hunit_-_Calenda%CC%81rio__image10_ede0453b.png)

## **Quando utilizar a Grade de Informações (Nova)**

Essa tela deve ser utilizada sempre que o hoteleiro precisar:

* Conferir se as informações foram corretamente enviadas aos canais
* Comparar tarifas e restrições entre canais
* Analisar o comportamento comercial por período
* Validar ajustes realizados recentemente

Ela é uma das principais ferramentas de acompanhamento e controle dentro do Hunit.

## **Glossário**

|  |  |
| --- | --- |
| **Termo** | **Descrição** |
| **Calendário** | Módulo do Hunit onde são gerenciadas tarifas, disponibilidades e restrições. |
| **Canal de venda** | Plataforma onde as diárias são comercializadas (Booking, Expedia, entre outros). |
| **Categoria** | Tipo de acomodação do hotel. |
| **Disponibilidade** | Quantidade de quartos disponíveis para venda. |
| **Grade de Informações (Nova)** | Tela do Hunit que exibe as informações enviadas aos canais, segmentadas por canal. |
| **Restrição** | Regra que limita a venda, como estadia mínima ou bloqueio de chegada e/ou saída. |
