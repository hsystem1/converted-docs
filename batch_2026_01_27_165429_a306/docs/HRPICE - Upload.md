**HRPICE – Upload**

## **O que é o Upload HPRICE**

O **Upload HPRICE** é uma funcionalidade do **produto HPRICE**, utilizada para **alimentar manualmente dados estratégicos de ocupação e indicadores de performance (KPIs)** da propriedade.

Essa funcionalidade é indicada para hotéis que **não possuem integração automática com PMS** ou que desejam **complementar, validar e manter um histórico organizado de informações operacionais e financeiras** dentro do HPRICE.

Ao manter esses dados atualizados, o gestor garante uma base sólida para análises de desempenho, ajustes de estratégia tarifária e acompanhamento da evolução do negócio.

## **Visão Geral da Funcionalidade**

O Upload HPRICE é dividido em dois grandes blocos:

* **Upload de Ocupação**
* **Upload de PMS KPI (Indicadores-chave de Performance)**

Cada bloco possui um objetivo específico, mas ambos trabalham de forma complementar para oferecer uma visão completa da performance da propriedade.

## **Objetivo da Funcionalidade**

### **Upload de Ocupação**

A tela de **Upload de Ocupação** permite carregar manualmente as informações de **ocupação atual e futura** do hotel.

Esse processo é necessário quando a propriedade **não possui integração com PMS**. Caso exista integração ativa, essas informações são recebidas automaticamente, não sendo necessário o preenchimento manual.

O principal objetivo é garantir que o HPRICE possua dados confiáveis de ocupação para apoiar análises, relatórios e estratégias de precificação.

### **Upload de PMS KPI (Key Performance Indicators)**

Os **KPIs (Indicadores-chave de Performance)** são métricas utilizadas para avaliar se o hotel está atingindo os objetivos operacionais e financeiros definidos.

Por meio dessa funcionalidade, o usuário pode importar dados consolidados do PMS para o HPRICE, criando um **histórico de evolução de performance**, mesmo sem integração direta.

Esses dados são fundamentais para análises estratégicas e acompanhamento de resultados ao longo do tempo.

**Por que utilizar o Upload HPRICE**

* A funcionalidade deve ser utilizada sempre que o hotel precisar:
* Carregar histórico de ocupação e reservas
* Importar dados de múltiplas datas de uma só vez
* Sincronizar informações provenientes de sistemas externos (PMS)
* Reduzir erros de digitação manual
* Garantir padronização dos dados
* Validar automaticamente as informações durante o processo de importação
* O Upload HPRICE contribui diretamente para a confiabilidade das análises realizadas dentro do produto.

O acesso às telas de upload varia conforme o tipo de informação que será carregada.

**Upload de Ocupação – Passo a Passo**

### **1. Baixar a planilha modelo**

* Antes de realizar qualquer envio, é necessário clicar em **Baixar planilha modelo**.
* Essa planilha já vem estruturada no formato correto, garantindo que o sistema consiga validar e importar os dados sem inconsistências.

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_165429_a306/images/HRPICE_-_Upload_image3_d9147989.png)

### **2. Preencher a planilha de ocupação**

Na planilha, o usuário deverá informar os dados solicitados, sempre respeitando:

* O formato correto das datas
* Os campos obrigatórios
* A estrutura original da planilha (sem alterar nomes de colunas)

Esses cuidados evitam erros durante o upload.

### **3. Frequência recomendada de upload**

Caso a propriedade utilize o **módulo de flutuação de tarifas do HPRICE**, recomenda-se realizar o upload da ocupação **duas vezes ao dia**.

Essa prática garante que o sistema utilize informações atualizadas para corrigir e ajustar automaticamente as tarifas.

### **4. Realizar o upload**

Após o preenchimento da planilha:

1. Selecione o arquivo na tela de Upload de Ocupação![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_165429_a306/images/HRPICE_-_Upload_image4_73719135.png)
2. Clique no botão **UPLOAD**![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_165429_a306/images/HRPICE_-_Upload_image2_3d680dfe.png)

O sistema fará a validação automática dos dados e, caso estejam corretos, concluirá a importação.

## **Upload de PMS KPI – Passo a Passo**

### **1. Baixar a planilha modelo**

Na seção **PMS KPI**, clique em **Baixar planilha modelo**.

Essa planilha é específica para indicadores de performance e contém campos próprios para preenchimento.

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_165429_a306/images/HRPICE_-_Upload_image6_36b63f64.png)

### **2. Preencher os dados obrigatórios**

Na planilha de PMS KPI, devem ser informados:

* Data
* Percentual de ocupação
* UH’s (Unidades Habitacionais vendidas)
* Diária média (ADR)
* RevPAR
* Receita do dia

Esses indicadores refletem o desempenho real da propriedade no período informado.

![Imagem](https://raw.githubusercontent.com/hsystem1/converted-docs/main/batch_2026_01_27_165429_a306/images/HRPICE_-_Upload_image7_9e1910a4.png)

### **3. Frequência de envio**

O upload de PMS KPI pode ser realizado:

* Diariamente
* Quinzenalmente
* Mensalmente

A frequência ideal depende da estratégia do hotel, mas o objetivo principal é **construir um histórico consistente de evolução de receita e performance**.

### **4. Realizar o upload**

Após preencher a planilha:

1. Selecione o arquivo correspondente
2. Clique em **UPLOAD**

O HPRICE validará automaticamente as informações antes de concluir a importação.

## **Importância do Upload HPRICE**

Manter os dados de ocupação e KPI atualizados é essencial para:

* Análises corretas de desempenho
* Ajustes estratégicos de tarifas
* Acompanhamento da evolução de receita
* Tomada de decisão baseada em dados reais

Mesmo propriedades com PMS integrado podem utilizar o Upload HPRICE como apoio para histórico e validação de informações.

## **Glossário**

|  |  |
| --- | --- |
| **Termo** | **Definição** |
| **HPRICE** | Produto voltado à gestão de preços, ocupação e estratégia tarifária. |
| **Ocupação** | Percentual ou volume de unidades ocupadas em determinado período. |
| **PMS (Property Management System)** | Sistema de gestão interna do hotel. |
| **KPI (Key Performance Indicator)** | Indicador-chave de performance. |
| **UH (Unidade Habitacional)** | Unidade de acomodação disponível para venda. |
| **ADR (Diária Média)** | Valor médio das diárias vendidas. |
| **RevPAR** | Receita por unidade disponível (Revenue per Available Room). |
